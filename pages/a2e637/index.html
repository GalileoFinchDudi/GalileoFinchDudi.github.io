        聊聊学习相关的

        作为混迹研发行业的一名小学生，我想，学习是一件非常重要的事情，也是一件非常有意思的事情。今天，就来聊聊学习相关的事情。如果荒废了太久，刚开始投入学习，千万不要给自己太大的压力。因为，学习是一件非常漫长的事情，不要想着一蹴而就，要从每次短小的学习中获得成就感，这样才能坚持下去。

        如果你是乐观开朗的人，可以试着把学习到的东西分享给别人，这样可以让你更加的有成就感，也可以让你更加的坚持下去。简称装X。

        如果你是一个内向的人，那么就把学习到的东西记录下来，这样可以让你更加的有成就感，也可以让你更加的坚持下去。前期可以以量来衡量自己的学习成果，后期再以质来衡量自己的学习成果。

        以下是我对学习的一些拙劣的理该，大纲如下：

        # 概览图

        # 英语

        英语是一门非常重要的语言，不仅仅是在研发领域。客观的来说，技术性的东西大多数都是从国外先发展推广的，如果不懂英语，就会错过很多东西。当然，如果你只是想在国内混口饭吃，那么英语对你来说可能并不是那么重要了。

        学习英语我常用的工具就是：

        工具

        anki，chrome-沙拉划词，扇贝

        程序员的英语可以不用太关注口语和听力，重点是阅读和写作，当然，如果你想去国外发展，或者参与一些remote job。那么口语和听力也是非常重要的。

        我会在chrome上阅读一些英文的技术文章，然后用沙拉划词来查看单词的意思，并一件选择加入到anki中，然后在anki中进行背诵。相对于anki，扇贝有着不错的社区环境，可以在里面找到更多的学习伙伴，可以参加同桌打卡或者是参加一些小组活动，这样可以让你更加的有成就感，也可以让你更加的坚持下去。

        素材来源其实有很多，比如：

        资源

        medium: 一些前沿的技术文章，免费版每个月可以阅读三篇。

        github: 一些开源项目的文档，也有一些不错的repo收录技术文章

        twitter: 一些大佬的推特，可以关注一些大佬，看看他们的推特

        翻译

        不要纠结于翻译的准确性，只要能够理解大概的意思就可以了。只需要做一个翻译搬运工，把一些好的文章翻译过来，然后发表在自己的博客上。

        我的方法是刚开始无法把整个点串起来的时候，就把每个点都翻译，记录到自己的anki或者扇贝里面，定期的刷。英文常见的单词实际上也就几千个，再加上一些技术性的单词，其实也就一万多个，所以，只要坚持下去，就一定可以看懂大部分的英文文章。

        手感

        每天花20分钟练练手感，可以通过访问qwerty (opens new window)多练习打字的手感，提高自己的手速。这个环节其实不用太可以追求单词的认识，只关注在不断的提高自己的英文打字速度就行，这样可以形成肌肉记忆，后续再确定输出的时候，就会非常快了。

        搭建自己的学习环境

        鸡汤

        工欲善其事必先利其器

        云效

        云效是阿里云出的一套提高研发能效的平台，包含了很多工作流程辅助的工具：项目写作，知识库，代码托管，流水线，制品库等。

        作为个人小团队，基本上所有一线大厂的流程都能接触到了，以专业的软件工程要求自己，不要怕麻烦，耽搁自己的技术学习时间。实际上这些内容的实操对自己的经验有很大的帮助。

        WSL(Windows subsystem for Linux)

        WSL是微软出的一套在windows上运行linux的工具，可以在windows上运行linux环境。这个是一个非常轻量的linux环境，我一般会把项目放在WSL环境中，这样的话可以屏蔽很多开发环境的差异，比如换了电脑，或者换了系统，都不会影响到项目的运行。

        另外就是重装很快，作为熟悉linux练手是一个不错的选择。

        网络环境

        作为一个程序员，学会科学上网是必须的。

        一台旧电脑

        效果等同于WSL，多用来练手如何优化系统，让系统更加的流畅。

        VPS(Virtual Private Server)

        常见的有阿里云，腾讯云，国外的有Vultr，DigitalOcean，Linode等。这些都是可以用来练手的，比如搭建一个博客，搭建一个网站，搭建一个git服务器等等。从而展开到学习域名，DNS，TLS/HTTPS等等。

        键盘

        这个就因人而异了，有些人可能喜欢用一些不错的键盘，提高自己编程的幸福感。

        # 养成学习习惯

        学习没有捷径，不动手永远无法成为高手。

        # 非持续性的学习

        先说一下非持续性的学习，也是普遍经历的场景。比如现网有个复杂的case，需要处理，这个时候可能花费一些时间去学习相关知识，做一下技术攻坚。另外就是面试突击，在面试前临时抱佛脚的学习一下，用于应付面试。这样的学习是非持续性的，短时间接触大量知识，是否能够吸收，就看个人的能力了。

        # 持续性的学习

        持续性的学习，就是每天都要学习一点，不要想着一蹴而就，要从每次短小的学习中获得成就感。

        # 学院派

        学院派的学习，就是按照一定的计划，系统的学习一门技术，比如学习CSAPP，DDIA这种大部头，就需要从头到尾的系统学习。

        学院派的学习想要完全掌握一个技术，需要花费大量的时间，但是学习的效果是非常好的，可以让你对一个技术有一个全面的认识。但是苦于没有合适的时间场景，可能会导致学习的效果不是很好，因为没有实践的场景，很多东西都是空中楼阁。而且长时间不去实践运用的话，很容易忘记。

        # 实用派

        实用派的学习，是基于目的学习。先表面再深入的学习，比如学习一个框架，先学习如何使用，然后再去深入的学习原理。这样的学习方式，可以让你快速的掌握一个技术。

        我曾经举过一个粗浅的例子，比如我们想要开车，刚开始的目标还是到达目的地，而不是去学习汽车的原理。当你真的上了上了赛道，需要提高自己的驾驶技术的时候，再去学习汽车的原理，这样会更加的容易理解。

        # 学习金字塔

        被动学习

        当你听到一些新的知识的时候，或者看到一些新知识的时候，这个时候你的大脑会自动的把这些知识存储到你的大脑中，这个过程就是被动学习。

        但是学习效果并不理想，因为你并没有主动的去学习，只是被动的接受了一些知识，这些知识并没有被你深入的理解，别人遇到的问题，当时绞尽脑汁去解决的问题，在你的视角并不深刻。

        主动学习

        实践和教都是最好的吸收方式，实践中不断的踩坑解决，反复的加深印象。教授别人时，不断地自省，不断的总结自然也就刻在脑子里了。

        # 总结

        学习真的是一件非常漫长的事情，要保持好自己的心态，从中发现乐趣，激发自驱力才能坚持下去。
